# Airi Lang (치사카 아이리 언어) 핵심 사용 가이드 🐺

Airi Lang은 치사카 아이리(Chisaka Airi)의 말버릇과 페이즈 커넥트의 밈들을 기반으로 작성된 에소테릭(Esoteric) C++ 트랜스파일 언어입니다. 아이리 언어로 머리를 가스라이팅 당할 준비가 되신 프로그래머들을 위해, 컴파일러 생태계의 주요 문법과 무시 조건(Tokenizer Rules)을 파훼하는 법을 설명합니다.

---

## 1. 토크나이저(Tokenizer) 분할 및 무시 규칙
컴파일러 렉서가 코드를 읽어들일 때, 일반적인 C++과 다르게 **"단어(Identifier)의 연속성"**과 **특수 밈 기호**를 우선적으로 판독합니다.

1. **연속된 문자는 식별자로 취급:** 
   `A`, `i`, `r`, `I` 기호들은 덧셈, 뺄셈, 곱셈, 나눗셈을 뜻하는 연산 기호입니다. 단, 이것들은 띄어쓰기(` `)를 통해 분절되지 않으면 하나의 영단어(예: `add`, `Airi`)로 뭉쳐져 식별자가 됩니다.
   - ⭕ **올바른 연산**: `x A y` (x 더하기 y)
   - ❌ **틀린 연산**: `xAy` (C++의 Identifier 'xAy' 하나로 인식되어 에러 뿜음)

2. **특수 기호 우선 판독 (`match` System):**
   `Ai...`(열린 소괄호), `...iriA`(닫힌 중괄호) 특수 기호들은 띄어쓰기가 없어도 고유 토큰으로 우선 분할됩니다.
   - 예시: `add(x)` 를 아이리어로 시도할 때 `addAi...x...ri` 로 적어도 `add`, `Ai...`, `x`, `...ri`로 안전하게 쪼개집니다. 

---

## 2. 함수와 반환 (Functions & PON)
함수를 작성할 때는, 언제 폰(PON)을 저지를지 모르는 아이리의 정신 상태를 형상화해야 합니다.

- **`Ai`**: 함수 선언의 시작 (C++의 `function` / 반환 타입 선언부 위치)
- **`PON`**: 반환문 (`return`). 어떤 값을 뱉든 불명예스럽게 쫓겨나는 것을 내포합니다.
- **`Airi>`** (`->`): 반환 타입 화살표 명시.

```cpp
// 예시: 10과 20을 더한 뒤 u32 자료형을 반환하는 함수
Ai add Ai... x .. Ai_32 . y .. Ai_32 ...ri Airi> Ai_32 Airi...
    PON x A y ,
...iriA
```
*해석: 함수(Ai) add ( x : u32 , y : u32 ) -> u32 { return x + y ; }*

## 3. 메모리와 변수 (Volatile & Initialization)
OS 개발까지 가능하도록 메모리 제어 규칙이 포함되어 있습니다. 
- 변수 선언 시 **`ri`** (`let`) 키워드를 사용합니다.
- 변수에 값을 부여할 땐 말끝을 흐리며 책임에 회피하는 **`...`** (대입, `=`)을 씁니다.

```cpp
// C++: let my_var = 100;
ri my_var ... 100 ,
```

## 4. 고급 문법: 가스라이팅(Gaslight)과 식민지 파싱
포인터나 타입을 맞추기 위해 강제로 C++ 컴파일러를 속일 때 사용합니다. 타입 캐스트나 하드웨어 포인터를 지정할 때 매우 적절합니다.

- **`Gaslight` (타입 캐스팅)**:
  컴파일러가 타입이 다르다고 반항(`PON! Type Mismatch`)할 때, 강제로 `AiriType`(포인터) 등 다른 타입으로 씌워버립니다.
- **`Gacha_` (메모리 주소 도박)**: 
  VGA 버퍼 등 위험한 하드웨어 주소를 나타내는 16진수(`0x`) 대신 `Gacha_` 를 씁니다.

```cpp
// C++: ptr<u16> vga_buffer = (ptr<u16>) 0xB8000;
ri vga_buffer .. AiriType Ai_16 ... Gaslight( Gacha_B8000 into AiriType Ai_16 ) ,
```

## 5. 타입과 논리 연산
- `Wolf`: 참 (`true`) - "나 늑대 맞아!"
- `woof`: 거짓 (`false`) - "Wolf not woof!" 
- `PON?`: 동일 여부 비교 (`==`) - "너 실수한 거 맞아?"
- `!PON?`: 부정 비교 (`!=`) - "실수 아니지?"

이제 여러분은 누구보다 당당하게 늑대의 정신으로 컴파일러를 다루며 에러를 발생시킬 수 있습니다!

## 6. 콘솔 출력 (Print)
화면에 글자나 계산 결과를 띄우고 싶다면 `phase_connects_famous_azooma` 키워드를 사용합니다.

```cpp
// 문자열 출력
phase_connects_famous_azooma "Hello, World!" ,

// 수식 결과 출력
phase_connects_famous_azooma add Ai... 10 . 20 ...ri ,
```
*해석: print("Hello, World!") / print(add(10, 20))*

문자열은 쌍따옴표(`"..."`)로 감싸서 표현합니다.

## 7. 스크립트 실행 방법 (Transpile & Run)

Airi Lang 컴파일러는 작성한 아이리어 스크립트를 읽어 C++ 코드로 자동 번역(Transpile)하고, 그 즉시 컴파일 및 실행까지 백그라운드에서 한 번에 처리합니다. 터미널(CLI) 환경에서 코드 실행은 다음 명령어 하나로 모두 해결됩니다.

```bash
# 기본 사용법: ./main_integrated [실행할 파일.airi]
./main_integrated my_code.airi
```

위 명령어를 치면 다음과 같은 자동화된 과정이 일어납니다:
1. `my_code.airi` 문법을 검사하고 파싱합니다.
2. `my_code.cpp` 라는 표준 C++ 소스 코드를 생성합니다.
3. `[Builder]` 모듈이 켜지며 `g++` 를 통해 `output_bin` 실행 파일로 컴파일합니다.
4. `[Runner]` 모듈이 켜지며 `./output_bin` 결과를 즉시 출력합니다.

코드 속에 C++의 메인 진입점을 뜻하는 `Ai main Ai... ...ri Airi> Ai_32 Airi...` 형태의 함수를 구축해두시면 이 모든 과정이 매끄럽게 연결됩니다!
