# 치사카 아이리(Chisaka Airi) 컨셉 난해한 프로그래밍 언어(Esoteric Language) 설계안

기존 `sulgye.txt`에 정의된 OS 개발 언어의 기초 뼈대(설계)는 완벽히 유지한 채, **페이즈 커넥트(Phase Connect) 소속 버튜버 치사카 아이리(Chisaka Airi)**의 특징적 밈과 유행어, 특유의 한숨 "Airi..."를 분해하여 문법(Syntax) 토큰으로 치환한 설계입니다. 

## User Review Required
> [!IMPORTANT]
> 언어의 백엔드와 AST 구조 자체는 그대로 C++ 타겟으로 유지되지만, 소스코드를 파싱하는 **Lexer의 키워드와 기호 인지 방식**이 아이리 컨셉으로 전면 교체됩니다. 제안드리는 밈(Meme) 구성과 아이디어들이 마음에 드시는지 확인해주시고, 추가하고 싶은 방송 밈이 있다면 피드백 부탁드립니다!

## Proposed Changes (아이리어 문법 규칙 선언)

### 1. "Airi" 기반 분해 기호 (Punctuation)
"Airi..." 라는 시청자들의 밈적 한숨과 어음을 분해하여 기존 기호를 완벽히 대체합니다.

| 기존 기호 | Airi Lang 기호 | 의미 / 설명 |
| :--- | :--- | :--- |
| `{` | `Airi...` | 컨텍스트의 시작. (깊은 한숨과 함께 블록 시작) |
| `}` | `...iriA` | 컨텍스트 끝. (Airi를 뒤집어서 블록 수렴) |
| `(` | `Ai...` | 소괄호 열기. |
| `)` | `...ri` | 소괄호 닫기. |
| `:` | `..` | 타입 지정 (점 2개). |
| `,` | `.` | 인자(Parameter) 구분자 (점 1개). |
| `;` | `,` | 구문(Statement)의 끝 (세미콜론 대신 쉼표 사용). |
| `->` | `Airi>` | 반환 타입 화살표. |

### 2. 치사카 아이리 세계관 키워드 매핑
기존 OS 개발용 시스템 키워드들을 아이리의 설정과 방송 밈으로 찰떡같이 교체합니다.

| 기존 키워드 | Airi Lang 키워드 | 유래 및 컨셉 |
| :--- | :--- | :--- |
| `fn` | `Ai` | 모든 함수의 시작, 즉 아이리의 앞글자. |
| `let` | `ri` | 변수 선언, 아이리의 뒷글자. |
| `struct` | `Airis` | **"Airi's Airis"** (가슴 네타). 수많은 데이터를 넉넉하게 담아둘 수 있는 거대한 구조체. |
| `return` | `PON` | **폰(PON)**을 저지르고 함수(방송)를 불명예스럽게 강제 종료시키며 값을 반환. |
| `interrupt` | `Airi!!!` | 시청자들이 경악하며 치는 채팅 그 자체. (인터럽트 발생!) |
| `asm` | `Phone_queen` | '폰퀸(실수투성이)'. 기계의 가장 깊은 곳을 건드리는 위험한 어셈블리 명령. |
| `mmio` | `Colony` | **"식민지 밈"**. 특정 하드웨어 메모리 주소(0xB8000 등)를 아이리의 제국(식민지)으로 강제 매핑 선언. |
| `volatile` | `Airhead` | 그녀의 덜렁거리는 텅 빈 머리(Airhead)처럼 값이 언제 어떻게 바뀔지 예측할 수 없는 메모리 상태. |

### 3. 타입(Type)과 불리언(Boolean)
타입은 `Ai`(부호 없는)와 `ri`(부호 있는)의 뒤에 붙는 **점(`.` )의 개수**로 메모리 비트를 나타냅니다.

* **정수형 타입**: 
  * `u8` -> `Ai.` / `u16` -> `Ai..` / `u32` -> `Ai...` / `u64` -> `Ai....`
  * `i8` -> `ri.` / `i16` -> `ri..` / `i32` -> `ri...` / `i64` -> `ri....`
* **포인터 (`ptr`)**: `AiriType`
* **불리언 (`bool`)**: 
  * `true` -> `Wolf` (본인은 늘 당당하게 늑대라고 긍정함)
  * `false` -> `woof` ("Wolf, not woof!" 밈. 개소리는 언제나 거짓이므로 false)

### 4. 연산자(Operators)
아이리의 스펠링 `A`, `i`, `r`, `I` 및 밈 문구를 활용합니다.

| 기존 연산 | 연산자 | 컨셉 |
| :--- | :--- | :--- |
| `=` (대입) | `...` | 값 부여 |
| `+` | `A` | 덧셈 |
| `-` | `i` | 뺄셈 |
| `*` | `r` | 곱셈 |
| `/` | `I` | 나눗셈 |
| `==` | `PON?` | "너 진짜 실수(PON)한 거 맞아?" (비교 연산) |
| `!=` | `!PON?` | "실수 안 했지?" |
| Type Cast | `Gaslight` | **가스라이팅 밈**. (예: `Gaslight(x into Ai...)`) 변수를 강제로 다른 타입이라고 컴파일러 정신을 가스라이팅함. |

### 5. 숫자(Numbers) 분해 규칙
사용자의 요청에 따라 기본 숫자는 `Airi...`를 쪼개서 **"이진수(Binary)"**로만 표현해야 합니다.
`Ai`는 `1`, `ri`는 `0`을 의미하며, 모든 숫자는 접두어 `Airi_` 로 시작해야 합니다.

* 숫자 `0` -> `Airi_ri` (0)
* 숫자 `1` -> `Airi_Ai` (1)
* 숫자 `5` -> `Airi_Ai ri Ai` (101)

**[🔥추가] 16진수(Hex) 압축 밈 기호 (`Gacha_`)**
포인터나 하드웨어 주소(예: 0xB8000)를 입력할 때 이진수 전개가 너무 길어지는 문제를 해결하기 위해, 아이리의 **"도박/가챠 중독"** 설정을 반영한 16진수 표기법을 지원합니다.
접두어 `Gacha_` 뒤에 일반 16진수를 입력합니다. (마치 가챠를 뽑듯 예측 불가한 메모리 주소를 뽑는다는 의미)
* `0xB8000` -> `Gacha_B8000`

### 6. 변수명 (Identifiers)
* 일반적인 **영문 스펠링을 허용**합니다. 가독성을 위해 완전히 제한하지 않으며, 기존 `add`, `x` 등을 자유롭게 쓸 수 있습니다.

## 💡 비교 예시 (C++ vs Airi Lang)

**[기존 코드 (`sulgye.txt` 기준)]**
```cpp
fn add(x: u32, y: u32) -> u32 {
    return x + y;
}

let vga: ptr<u16> = 0xB8000;
```

**[Airi Lang 변환 후]**
```cpp
Ai add Ai... x .. Ai... . y .. Ai... ...ri Airi> Ai... Airi...
    PON x A y ,
...iriA

ri vga .. AiriType Ai.. ... Gacha_B8000 ,   // (압축된 16진수 가챠 기호 사용)
```

## Verification Plan
피드백이 확정되면 다음의 후속 작업을 진행합니다.
1. `Lexer.h` / `Lexer.cpp` 의 토큰 파싱 정규식(Token rules)을 위 밈 규칙으로 완전히 오버라이드.
2. Syntax 파서가 `Airi...`, `PON` 키워드를 만나도 기존 AST(Abstract Syntax Tree) 구조로 문제없이 변환되는지 테스트 파일 컴파일 확인.
3. 에러 발생 시 `PON! Airi messed up at line X` 의 형태로 커스텀 콘솔 출력 로직 추가.
